# Canonical Sequence

Use this sequence when freezing architecture before implementation.

## Sections

1. `Vision`
   - What system are we building and why?
2. `Sync With User`
   - What does the user actually want? What does success mean?
3. `IFR`
   - What is the ideal final behavior?
4. `Reality Map`
   - What exists? What works? What is broken or unclear?
5. `Contradictions`
   - Where does the system pull in opposing directions?
6. `Resolution Principles`
   - By what laws will decisions be made?
7. `Options / Best Path`
   - What paths exist and which one is chosen?
8. `System Structure`
   - What are the artifact roles and architecture parts?
9. `Operational Model`
   - How should the system function as a workflow?
10. `Implementation Criteria`
   - What must be true before implementation is allowed, and what defines done?
11. `Prototype / Pilot`
   - Is a prototype needed, and what may it validate?
12. `Implementation`
   - What gets built, in what controlled order?
13. `Testing + Verification`
   - How will correctness be proven?
14. `Iteration / Rollout`
   - How does the system get refined and safely expanded?

## Session Rule

The active session file is the working ledger for this sequence.

Use canonical documents only after the path is frozen.
