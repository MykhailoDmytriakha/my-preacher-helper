### F5. Прочие residuals (не блокеры)

- **Кэш конфига дефолтов** — `config/aiModelDefaults` читается на каждый текст-вызов → process-level memo с коротким TTL (инвалидация на admin-save).

### F7. Grace-band лимит usage — «по благодати» (владелец 2026-07-14)
**Идея.** Не жёсткий блок ровно на лимите, а МЯГКИЙ порог с грацией. Мотив: одно легитимное действие может стоить >1 (транскрибация = STT + полировка = 2 списания), и падать на полпути — плохой UX; дух приложения (помощник проповедника) → благодать уместна как продуктовая черта, не только техника.
**Поведение:**
- Действие, начатое в пределах лимита, **ДОигрывается целиком** (даже если перескочит на 1-2 — напр. 99 → 101).
- Мягкий блок **НОВЫХ** действий, когда лимит достигнут/чуть превышен.
- **Hard-cap** на бо́льшем пороге (напр. `limit + grace`, где grace = 10% или фикс) — дальше реально нельзя запустить новое.
- При перерасходе в настройках/админке — тёплое сообщение + **СТИХ про благодать** (напр.: «даже если мир нечестен — будьте честны и живите по благодати» / «Христос простил нам все долги»). Ротация нескольких стихов.
**Связь.** Заменяет строгий атомарный фикс **B9/TOCTOU** (`BUGS.md`) и закрывает старый residual F5 «TOCTOU soft-cap»: малый overshoot допустим **by design** → атомарность не критична; остаётся только hard-cap + grace-message. Реализация: soft limit (allow до `limit+grace`) + hard limit + UI перерасхода со стихом.

### F6. Referral anti-abuse — human-in-the-loop (дизайн владельца, 2026-07-14) — заменяет F5-Sybil
**Не авто-блок, а флаг + ручной разбор:**
- Первые **2 реферала** инвайтера — авто-грант без проверки (как сейчас).
- На **3-м реферале**: (а) флаг `referralWarning` на инвайтере (виден в админке); (б) письмо на `OWNER_EMAIL` «проверь юзера на честность» + контекст (uid · сколько рефералов · кого пригласил).
- **Промо продолжает начисляться** — человек МОЖЕТ законно набрать период; флаг = сигнал на ручной разбор, не авто-стоп.
- **Админка — новый контроль:** «отключить / аннулировать промо» у юзера **+ показывать активность** флагнутого (кол-во проповедей · `last_seen_at` · usage) — чтобы отличить «живого» от «пустышки» одним взглядом.
**Философия приёмки (для ручного суждения):** набрал + пишет проповеди = ок · набрал + ничего не делает = подозрительно · делает глупости = предупреждение → блок. Данные уже есть (`last_seen_at`, счётчики).
**Механика для справки (`referral.server.ts`):** награда = **+30 дней tier1 за КАЖДОГО реально пришедшего, кумулятивно**. Защита сейчас: `email_verified` + аккаунт <24ч + один клейм на invitee. (20 приглашений, 2 регистрации → 60 дней.)

## Concept Map / Исследовательские связи    

- Пользователь исследует тему (например, "Дети Соломона") и делает заметку (вопрос-ответ).
- Затем находит связанную информацию ("Кто мать Ровоама?") и делает вторую заметку, связывая её с первой.
- Постепенно строится граф связей (Concept Map) без единого центрального узла (в отличие от классического Mind Map).
- Каждое новое исследование (вопрос-ответ-заметка) становится узлом в этой карте.
- Это позволяет видеть динамику исследования и переходить от одной темы к другой (например, от Соломона к Ровоаму) через выявленные связи.
- Визуально это должно выглядеть как сеть связанных фрагментов знаний, которую можно дополнять и исследовать.
- Суть в том, что пользователь сам строит эти связи по мере изучения материалов.
